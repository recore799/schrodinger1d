Crear — Extiende el proyecto

En este nivel se invita a los estudiantes y colaboradores a modificar y ampliar el código del proyecto schrodinger1d . El propósito es fomentar la creatividad, la exploración y el aprendizaje a través de la experimentación directa con los módulos existentes.

Clonar el repositorio

Para comenzar, clona el repositorio en tu computadora:

git clone https://github.com/recore799/schrodinger1d.git
cd schrodinger1d

El proyecto está escrito en Python y organizado en módulos independientes. Se recomienda crear un entorno virtual e instalar las dependencias: 

python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

Nota: El repositorio es privado; si deseas contribuir directamente, deberás solicitar acceso como colaborador o hacer un fork en tu propia cuenta para trabajar en una copia del proyecto.

Ideas para extender el módulo de Hartree–Fock

  • Nuevas integrales: crear rutinas que calculen integrales de dos electrones incluyendo orbitales p.
  • Mejor convergencia SCF: implementar métodos de mezcla o aceleración DIIS.
  • Densidad inicial mejorada: probar estrategias alternativas para construir la matriz de densidad inicial \( P \).
  • Comparaciones de referencia: integrar scripts que validen los resultados contra programas como PySCF o Psi4.

Ideas para extender el módulo de Numerov

  • Nuevos potenciales: implementar funciones como \( V(x)=ax^4+bx^2 \) o \( V(x)=-V_0 e^{-\alpha x^2} \).
  • Estados excitados: desarrollar rutinas para encontrar y normalizar estados con nodos.
  • Visualización: integrar matplotlib o plotly para generar gráficas interactivas.
  • Comparaciones analíticas: verificar la precisión frente a soluciones conocidas del oscilador armónico y el átomo de hidrógeno.

Cómo contribuir

Para añadir una nueva característica o mejora, crea una rama y envía tus cambios:

git checkout -b feature/nueva_funcionalidad
git commit -am "Implementa nueva rutina de integrales"
git push origin feature/nueva_funcionalidad

Incluye comentarios claros, pruebas básicas y una breve descripción en el README.md. Las contribuciones pueden enviarse como pull requests.
