Crear — Extensión y experimentación

En la etapa de Crear el participante transforma la comprensión en producción científica: deja de ejecutar ejemplos existentes para diseñar, modificar y experimentar con nuevos algoritmos. Aquí se integran todos los conocimientos previos —físicos, matemáticos y computacionales— para generar ideas originales.

Crear es el punto más alto del aprendizaje: implica dominio técnico, juicio crítico y capacidad de construir algo nuevo a partir de lo comprendido.

Este proyecto fue concebido como un entorno abierto. Si deseas comenzar a colaborar, puedes clonar el repositorio directamente desde GitHub:

Ejemplos de experimentación

Estos ejemplos muestran cómo introducir nuevas ideas dentro del código existente. Cada uno modifica un aspecto fundamental del modelo —potencial, integración o condiciones iniciales— para explorar sus consecuencias físicas.

1. Probar un nuevo potencial efectivo

from src.numerov.numerov import solve_atom

Z = 1
def custom_potential(r):
    # Potencial de Coulomb más término cuadrático
    return -Z / r + 0.002 * r**2

r, psi = solve_atom(n=1, l=0, potential=custom_potential)

Este cambio introduce un término confinante débil que altera el espectro de energía. Analizar su efecto permite conectar la física atómica con modelos de confinamiento cuántico.

2. Implementar un nuevo esquema de integración

import numpy as np

def numerov_adaptativo(V, E, x, dx):
    """Versión experimental del método de Numerov con paso variable."""
    y = np.zeros_like(x)
    y[0], y[1] = 0.0, dx
    for i in range(1, len(x)-1):
        dx_local = dx * (1 + 0.001 * abs(V[i]))
        g = 2*(V[i]-E)
        y[i+1] = (2*y[i]*(1 - 5*dx_local**2*g/12) - 
                  y[i-1]*(1 + dx_local**2*g/12)) / (1 + dx_local**2*g/12)
    return y

Este ejemplo explora una versión adaptativa del método de Numerov, ajustando el paso de integración en función del potencial local para mejorar la estabilidad numérica.

3. Definir una nueva matriz de densidad inicial en Hartree–Fock

import numpy as np

def guess_density_random(nbasis, seed=42):
    """Matriz de densidad inicial aleatoria normalizada."""
    np.random.seed(seed)
    P = np.random.rand(nbasis, nbasis)
    P = 0.5 * (P + P.T)  # Simetrizar
    P /= np.trace(P)
    return P

Un esquema de arranque alternativo puede acelerar o diversificar la convergencia del procedimiento SCF. Esto resulta útil para estudiar la sensibilidad del método Hartree–Fock a las condiciones iniciales.

Guía rápida de colaboración

El proyecto schrodinger1d está pensado para crecer de forma colectiva. Cualquier estudiante o investigador puede proponer mejoras, nuevos modelos o ejemplos pedagógicos. A continuación se resumen las prácticas recomendadas para contribuir eficazmente:

  • Estructura de trabajo: coloca tus scripts o notebooks en la carpeta examples/ con un nombre descriptivo y claro. Evita sobreescribir archivos existentes.
  • Documentación: registra los cambios realizados en README.md o crea un archivo de notas dentro de docs/ que describa tu enfoque y resultados.
  • Pruebas iniciales: valida tus modificaciones con sistemas sencillos (oscilador armónico o hidrógeno) antes de extenderlas a casos más complejos.
  • Colaboración formal: cuando tu código sea estable y replicable, abre un pull request en GitHub describiendo de forma breve y técnica qué problema aborda y cómo lo soluciona.
  • Motivación científica: explica la motivación física, numérica o pedagógica detrás de tu contribución. Una buena idea siempre parte de una pregunta bien planteada.
  • Estilo y claridad: sigue la estructura del código existente, usa comentarios breves y consistentes, y prioriza la legibilidad sobre la optimización prematura.

Las contribuciones bien documentadas no solo amplían la librería, sino que también fortalecen su valor educativo y colaborativo.

Objetivos de la etapa

La etapa de Crear culmina el recorrido de la taxonomía, donde la comprensión se convierte en producción. Aquí se busca fomentar la autonomía investigativa y la curiosidad computacional.

  • Aprender a modificar y extender la base de código existente con criterio científico.
  • Probar nuevas ideas: modelos de potencial, rutinas de integración o estrategias SCF.
  • Evaluar la sensibilidad del algoritmo a las condiciones iniciales y a los parámetros numéricos.
  • Aplicar pensamiento crítico para diseñar, justificar y validar mejoras concretas.
  • Desarrollar habilidades colaborativas propias del trabajo científico moderno.

Crear no solo implica escribir código: es un acto de exploración intelectual. Cada nuevo módulo o idea añade una pieza al mapa colectivo de la física computacional.
